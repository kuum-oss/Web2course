## МЕТА РОБОТИ

Ознайомитись із можливостями каскадних таблиць стилів (CSS) для створення користувацьких лічильників та стилізації елементів веб-форм. Набути практичних навичок оформлення полів введення, списків вибору та текстових областей відповідно до сучасних принципів UI/UX дизайну.

---

## ХІД РОБОТИ

Під час виконання лабораторної роботи макет веб-сторінки **«Premium Art Gallery»** було доповнено двома функціональними секціями:

* блоком **«Як відбувається візит»**, у якому використано CSS-лічильники;
* блоком **«Забронювати квиток»**, що реалізований за допомогою HTML-форми та повністю стилізований засобами CSS.

Дизайн сторінки виконано у темній преміальній кольоровій схемі з використанням золотих акцентів. Основні CSS-властивості, застосовані під час розробки, наведено у таблиці.

### Таблиця 1 — CSS-властивості для роботи з лічильниками та формами

| Властивість          | Опис застосування у проєкті                                     |
| -------------------- | --------------------------------------------------------------- |
| `counter-reset`      | Створення нового CSS-лічильника у батьківському контейнері      |
| `counter-increment`  | Автоматичне збільшення значення лічильника для кожного елемента |
| `content: counter()` | Виведення значення лічильника через псевдоелемент               |
| `outline: none`      | Видалення стандартної рамки браузера при фокусі                 |
| `resize: vertical`   | Дозволяє змінювати розмір textarea лише по вертикалі            |

### 1. Використання CSS-лічильників

Для нумерації етапів відвідування було відмовлено від стандартного тегу `<ol>`, оскільки він обмежує можливості стилізації. Натомість застосовано CSS-лічильники, що дозволили створити великі акцентні номери (01, 02, 03), які частково виходять за межі карток.

```css
/* Ініціалізація лічильника */
.steps-grid {
    counter-reset: step-counter;
}

/* Формування номера через псевдоелемент */
.step-card::before {
    counter-increment: step-counter;
    content: "0" counter(step-counter);
    
    position: absolute;
    top: -25px;
    background-color: var(--bg-main);
    color: var(--gold-accent);
    border: 2px solid var(--gold-accent);
}
```

Застосування цього підходу дозволяє автоматично перенумеровувати блоки при їх додаванні або зміні порядку без редагування HTML-коду.

---

### 2. Стилізація веб-форм

Стандартні стилі браузера для елементів форм було перевизначено з метою збереження єдиного дизайну сторінки. Особливу увагу приділено стану фокусу (`:focus`), щоб користувач чітко бачив активне поле введення.

```css
/* Базові стилі полів */
input[type="text"], input[type="email"], select, textarea {
    width: 100%;
    padding: 15px;
    background-color: rgba(255, 255, 255, 0.03);
    border: 1px solid rgba(255, 255, 255, 0.1);
    color: var(--text-light);
}

/* Ефект фокусу */
input:focus, select:focus, textarea:focus {
    outline: none;
    border-color: var(--gold-accent);
    box-shadow: 0 0 10px rgba(212, 175, 55, 0.2);
}
```

Завдяки цьому форма гармонійно інтегрується у дизайн сторінки та забезпечує зрозумілий візуальний зворотний зв’язок для користувача.

## ВИСНОВКИ

У результаті виконання лабораторної роботи було реалізовано стилізовані веб-форми та користувацькі списки з автоматичною нумерацією.

1. **Гнучкість CSS-лічильників:** використання `counter-reset` та `counter-increment` у поєднанні з псевдоелементами дозволяє створювати складні дизайнерські списки без обмежень стандартного HTML.
2. **Покращення UX форм:** перевизначення стандартних стилів браузера дозволило зробити форму частиною загального дизайну сайту та уникнути візуальних конфліктів.
3. **Зворотний зв’язок для користувача:** налаштування стану `:focus` підвищує доступність інтерфейсу та спрощує взаємодію з формою.

Отже, поставлену мету лабораторної роботи досягнуто. Створені елементи відповідають сучасним вимогам веб-дизайну, забезпечують зручність використання та мають цілісний візуальний стиль.
