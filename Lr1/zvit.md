## 2. Вступ

**Напрям дослідження:** Проєктування та розробка клієнтської частини (Frontend) веб-додатків з використанням сучасних технологій розмітки та стилізації (HTML5, CSS3).

**Мета навчальної практики:** Розробити сучасну, адаптивну та інтерактивну веб-сторінку, яка презентує інформацію про природні ландшафтні парки України та світу, закріпити навички роботи з каскадними таблицями стилів (CSS), семантичною версткою (HTML) та методологіями побудови сіток (Flexbox, CSS Grid).

**Завдання навчальної практики:**

1. Спроєктувати структуру веб-сторінки та розробити семантичний HTML-каркас.
2. Підібрати та інтегрувати медіа-контент (зображення високої якості).
3. Розробити файл стилів `style.css` з використанням сучасних UI/UX рішень (Glassmorphism, плавні анімації, тіні).
4. Забезпечити адаптивність веб-сторінки для мобільних пристроїв.
5. Провести тестування (відлагодження коду) та усунути синтаксичні помилки.

## 3. Лаконічний опис результатів виконання лабораторних робіт

В ході виконання роботи було створено повноцінний односторінковий лендінг. Роботу поділено на такі етапи:

* **Створення базової структури:** Сформовано файл `index.html`, де виділено логічні блоки: головний банер (`<header>`), секція інтро, секція українських парків, секція світових парків (`<section>`) та підвал (`<footer>`).
* **Стилізація елементів:** У файлі `style.css` підключено сучасний шрифт *Inter*, налаштовано змінні (`:root`) для кольорової палітри. Застосовано технологію CSS Grid для вирівнювання карток парків.
* **Інтерактивність:** Додано мікроанімації наведення (`hover`) на картки, що включають масштабування зображень (`transform: scale`) та появу об'ємних тіней.
* **Виправлення помилок (Дебагінг):** Виявлено та виправлено проблеми з незакритими тегами `</div>` та `</article>`, а також вирішено проблему блокування зображень (CORS) шляхом заміни джерел на надійні CDN (Unsplash, Pexels).

## 4. Текст програми (з коментарями)

Нижче наведено ключові фрагменти розробленого коду з детальними поясненнями.

**Фрагмент HTML-коду (Структура картки та сітки):**

```html
<div class="grid-container">
    
    <article class="park-card">
        
        <div class="card-image-wrap">
            <img src="https://images.unsplash.com/photo-1476514525535-07fb3b4ae5f1?q=80&w=800" alt="Озеро Синевир" class="card-img">
            <span class="card-badge">Карпати</span>
        </div>
        
        <div class="card-content">
            <h3>НПП «Синевир»</h3>
            <p>Найбільше та найглибше високогірне озеро України...</p>
            <a href="https://uk.wikipedia.org/wiki/Синевир_(озеро)" target="_blank" class="read-more-btn">Дізнатися більше</a>
        </div>
        
    </article>
</div>
```

**Фрагмент CSS-коду (Анімації та змінні):**

```css
/* Оголошення глобальних CSS-змінних для зручної зміни теми в майбутньому */
:root {
    --primary: #059669;       /* Основний акцентний колір (зелений) */
    --surface: #FFFFFF;       /* Колір фону карток */
    --transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1); /* Налаштування плавної анімації */
}

/* Базові стилі для картки */
.park-card {
    background-color: var(--surface);
    border-radius: 20px;     /* Заокруглення кутів */
    overflow: hidden;        /* Обрізання контенту, що виходить за межі заокруглень */
    box-shadow: 0 4px 6px rgba(0, 0, 0, 0.05); /* Легка тінь за замовчуванням */
    transition: var(--transition);
}

/* Ефект при наведенні курсору миші на картку (hover) */
.park-card:hover {
    transform: translateY(-8px); /* Картка "підстрибує" вгору на 8 пікселів */
    box-shadow: 0 20px 25px rgba(0, 0, 0, 0.1); /* Тінь стає більшою та розмитішою */
}

/* Плавне збільшення картинки при наведенні на батьківську картку */
.park-card:hover .card-img {
    transform: scale(1.08); /* Масштабування на 8% */
}
```

## 6. Змістовний аналіз отриманих результатів та висновки

**Аналіз результатів проектування та тестування:**
Під час розробки інформаційної системи використовувався підхід Mobile-First та принципи семантичної верстки. Використання властивості `display: grid` для сітки карток дозволило досягти математично точного вирівнювання елементів без використання застарілих методів (таблиць або `float`).

У процесі тестування було виявлено типові проблеми верстки:

1. **Порушення ієрархії DOM-дерева:** через відсутність закриваючого тегу `</div>` у блоці `.card-image-wrap` та помилку у написанні тегу `</article>`. Ці помилки викликали «злам» сітки. Після їх усунення браузер почав коректно рендерити контейнери.
2. **Захист від hotlinking (CORS):** зображення з ресурсів Вікіпедії не завантажувалися через локальний сервер середовища розробки. Проблему вирішено шляхом підбору альтернативних, відкритих для розробників джерел (Unsplash/Pexels).

**Висновки:**
В результаті виконання навчальної практики розроблено сучасну веб-сторінку, яка повністю відповідає поставленим завданням. Програма (HTML/CSS код) не містить синтаксичних помилок, успішно проходить валідацію і демонструє високу продуктивність. Застосування сучасного UI-дизайну (шрифт Inter, м'які тіні, анімації `hover`, скляний ефект) зробило інтерфейс інтуїтивно зрозумілим та естетично привабливим для кінцевого користувача. Мета практики досягнута в повному обсязі.
